# 다중 코스는 "원하는 길 선택(다양성)"이며 매번 무작위 방향으로 생성한다

코스를 2개 제공하는 목적은 **"원하는 길을 고르는 다양성"**이다 — 거리나 지형 차이가 아니다. 목표 칼로리를 정하면 필요한 거리가 거의 고정되므로 거리는 의미 있는 차별점이 아니고, 방위("북동쪽")도 사용자에겐 무의미하다. 실제 선택은 **지도(경로 모양·주변)** 를 보고 이뤄진다.

따라서: 요청마다 **무작위 방향 2개**를 뽑되 두 방향을 서로 90~270° 벌려 겹치지 않게 하고, 강·막다른 길 등 경로 생성이 실패하는 방향은 새 무작위 방향으로 재시도한다. 같은 출발지에서도 "갈 때마다 다른 코스"가 나온다("경로 다시 추천" 버튼 포함). 거리·지형을 암시하던 가짜 태그(짧은 거리/완만 등)는 제거했다.

## Considered Options

- **무작위 방향 + 지도 선택(채택)** — 다양성 목적에 부합, 갈 때마다 새 코스. 라벨 없이 지도로 고른다.
- **거리로 차별화(거부)** — 목표 칼로리가 거리를 고정하므로 의미 없는 가짜 차이.
- **방위 라벨(거부)** — "북동쪽 코스"는 사용자 선택에 도움 안 됨.
- **경사로 차별화(보류)** — "평지/언덕"은 의미 있으나 Elevation API 비용. 검증 후.
- **랜드마크 이름(보류)** — "○○공원 방면"은 의미 있으나 역지오코딩/Places 비용.
