# 완수 루프는 로그인 기능, 익명 단계는 전환 훅으로 둔다

ADR-0004는 "익명 우선 → streak 저장 시 로그인 유도 + 기록 병합"을 정했다. 구현을 보니 익명 완수 루프를 그대로 돌리기엔 비용 제약이 걸린다: 루프는 본질적으로 반복(끼니·날마다 분석)인데, 분석(analyze)이 OpenAI 호출이라 돈이 들어 게스트는 체험 1회로 제한된다. 따라서 게스트는 계획이 평생 1개뿐이라 여러 날 streak 같은 루프의 진짜 가치를 익명으로는 보여줄 수 없다.

그래서 익명 단계의 목표를 "루프 체험"이 아니라 **"강력한 한 입 → 로그인 전환"**으로 잡는다. 분석 1회 무료는 그대로 두어 비용을 통제하고, 그 1회 직후 **완수 루프의 가치(매일 streak·완수율·기록)를 보여주며 로그인을 유도**한다. 게스트의 1개 계획은 로그인 시 자동 이관된다(migrate, 이미 구현·로그인 시 호출됨). 진짜 완수 루프는 로그인 이후의 기능으로 둔다.

분석을 게스트에게 더 여는 선택(비용 감수)은 검증 초기·솔로 운영에 위험하므로 채택하지 않는다.

## Considered Options

- **루프는 로그인 기능 + 익명은 전환 훅(채택)** — 분석 비용 통제 유지, 작업도 작음(전환 경험 + 이관은 이미 됨).
- **분석을 게스트에 개방(거부)** — 익명 경험은 두꺼워지나 OpenAI 비용 발생. 초기엔 위험.
- **익명 완수를 localStorage로(거부)** — 비용 0이나 기기 변경 시 소실 + 서버 기반과 이원화.
