# 경로 따라가기는 포그라운드 실시간 추적(v1)으로 구현한다

앱이 코스를 "계획"만 하고 뛰는 동안엔 동행하지 않아, 사용자가 "어디쯤 왔나 / 잘 따라가고 있나"를 앱 안에서 확인할 수 없었다(정적 경로 + 외부 지도 링크뿐). 이를 **포그라운드 실시간 추적**으로 해결한다 — ADR-0001에서 "GPS 자동은 향상 레이어"라 미룬 것을, 백그라운드가 아닌 *포그라운드(화면 켜고 앱 열어둔 상태)* 방식으로 지금 실현하는 것이라 기존 결정과 정합한다.

**v1 범위:** "따라가기 시작" 버튼 → `watchPosition`(고정밀) + Wake Lock(화면 유지) → 지도에 현재 위치 마커 실시간 표시 + 이동 거리 누적. "중지"로 종료. **완수는 여전히 수동**(기록 화면). "잘 따라가고 있나"는 마커가 경로 선 위에 있는지 **눈으로** 확인(이탈 자동 감지는 후속).

**제약:** 포그라운드 전용(화면 끄면 웹은 위치 갱신 중단). Wake Lock·watchPosition은 iOS 사파리 지원이 제한적이라, 미지원 시 wake lock 실패는 조용히 무시하고 추적만이라도 동작하게 한다.

## Considered Options

- **포그라운드 실시간 추적 v1(채택)** — "어디쯤 왔나"를 지금(웹) 해결. 자동 완수·실측 칼로리의 토대.
- **자동 완수(지오펜스)까지 v1에 포함(보류)** — 왕복 완주 감지가 복잡. 추적이 도는 뒤 후속.
- **실측 거리 기반 실제 칼로리(보류)** — ADR-0007의 "정확도는 사후 측정"을 실현하나 범위 큼. 후속.
- **외부 Google 지도에 위임(현행)** — 앱을 벗어나고 완수 연결 끊김.
