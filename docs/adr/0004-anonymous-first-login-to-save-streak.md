# 익명 우선, 연속 기록이 쌓이면 로그인으로 저장 유도

이 제품엔 정체성 시스템이 둘 공존한다: 익명 쿠키(`eat_run_uid`, 현재 기록이 묶임)와 Supabase 로그인(OTP+OAuth, 이미 구현됐으나 로그인할 동기가 약함). 연속 기록(Streak)을 도입하면 둘의 충돌이 표면화된다 — streak은 정체성에 묶인 성취인데, 익명 쿠키는 쿠키 삭제·기기 변경으로 소실되기 때문이다.

입구의 즉시성(가입 없이 사진 찍고 분석)과 후크의 견고함(streak)을 둘 다 지키기 위해: **입구는 익명 쿠키로 유지**하고, **streak이 쌓인 뒤 "진척을 지키려면 로그인" 으로 전환을 유도**한다. 로그인이 차가운 장벽이 아니라 진척을 지키는 가치 있는 순간이 되며, 기존 로그인 기능에 비로소 동기가 생긴다. 로그인 시 **익명 쿠키의 기존 기록을 계정으로 병합(이관)**해 두 정체성 시스템을 잇는다.

## Considered Options

- **익명만(거부)** — 입구 마찰 0이나 streak이 쉽게 소실돼 장기 후크로 약함.
- **로그인 필수(거부)** — streak은 견고하나 가입 장벽이 입구 가치(즉시성)를 죽임.
- **익명 우선 → streak 저장 시 로그인 + 기록 병합(채택)** — 둘 다 충족. 단, 익명→계정 이관 로직이 필요.
