# 완수(Completion)는 수동 체크를 기본으로 한다

제품의 핵심 성공 지표는 완수율(Plan → Completion 전환율)이고, 완수를 데이터 모델에 도입해 다리(Bridge)의 반대쪽 끝을 닫는 것이 다음 발전 방향이다. 원래 구상은 GPS 실시간 위치로 경로 종료점 도달 시 자동 완료 처리하는 것이었으나, 웹은 백그라운드 위치 추적이 불가능(화면 끄면 JS 정지)해 "운동 내내 화면을 켜고 있어야 한다"는 제약에 막혀 다리가 영영 닫히지 않았다.

따라서 **완수는 우선 사용자가 직접 누르는 수동 체크로 닫는다.** 이것만으로 완수 데이터와 완수율 측정 토대가 완성된다. GPS 자동 완료 판정(포그라운드 `watchPosition` + 종료점 지오펜스 + wake lock)은 *없으면 제품이 안 도는 필수가 아니라* 수동 체크 위에 얹는 향상 레이어로 후순위에 둔다.

## Considered Options

- **GPS 자동 완료(필수)** — 마찰 0, 신뢰도 높음. 그러나 웹 백그라운드 한계로 "화면 상시 켜기"를 강제, 완벽을 노리다 출시 자체가 막힘. → 향상 레이어로 강등.
- **수동 체크(채택)** — 어디서나 동작, 구현 사소함. 정직성 의존·재방문 필요는 동기 레버로 별도 해결.
