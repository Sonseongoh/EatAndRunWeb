# PWA는 "설치형 셸"까지만, 푸시·오프라인 데이터는 제외한다

ADR-0003은 "웹/PWA로 완수 루프를 먼저 검증"하도록 정했고, PWA를 웹-우선 검증의 수단으로 명시했다. 그 일부로 PWA 패키징을 도입하되, 범위를 최소 설치형 셸로 한정한다. 목적은 "오프라인"이 아니라 **설치형(홈 화면)으로 재방문을 쉽게 만들어 완수 루프의 재참여 레버(ADR-0002)를 강화**하는 것이다.

**범위:**
- 웹 매니페스트 + 아이콘(192·512) + 서비스워커(설치 가능 요건 충족 + 정적 앱 셸 캐싱). 데이터는 항상 네트워크.
- **오프라인 데이터 캐싱은 하지 않는다.** 이 앱은 Supabase·OpenAI 의존이라 오프라인 가치가 작고, 캐시 정합성·오프라인 쓰기 큐가 완수 토글(쓰기)과 충돌할 위험이 크다.
- **푸시 알림은 하지 않는다.** 재참여는 푸시가 아니라 재방문 직면+streak으로 한다는 ADR-0002를 유지한다. PWA 설치가 푸시를 *가능케* 하지만, 그 결정은 별도다.

**설치 발견성:** 브라우저 기본 설치는 발견성이 나쁘다(안드로이드 메뉴 속 "앱 설치", iOS는 자동 프롬프트 없음). 그래서 **최소한의 커스텀 안내**를 둔다: 전역 하단의 닫을 수 있는 "앱 설치" 배너 1회 노출. 탭하면 지원 브라우저는 `beforeinstallprompt`로 네이티브 설치를, iOS는 "공유 → 홈 화면에 추가" 안내를 표시. 닫으면 기억하고(localStorage), 이미 설치(standalone)면 숨긴다. 타이밍 최적화·A/B 같은 캠페인은 검증 후로 미룬다.

## Considered Options

- **설치형 셸만(채택)** — ADR-0003·0001·0002 모두와 정렬, 단순.
- **오프라인 데이터 캐싱(거부)** — 가치 작고 캐시 정합성·오프라인 쓰기 큐가 완수 토글과 충돌 위험.
- **웹푸시 포함(보류)** — ADR-0002를 재검토해야 하는 별도 결정.
- **브라우저 기본 설치만(거부)** — 발견성이 나빠 아무도 설치 안 함 → PWA 목적(재방문) 무산.
