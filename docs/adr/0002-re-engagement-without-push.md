# 재참여(완수 유도)는 푸시 알림이 아니라 재방문 시점에 건다

완수율을 높이려면 사용자를 다시 데려와 완료를 누르게 해야 한다. 습관 앱의 통상적 해법은 푸시 리마인드지만, 이 제품은 웹 + 익명 쿠키 기반이라 푸시가 신뢰할 수 없는 채널이다(iOS 사파리는 PWA 설치 후에만 푸시 가능, 익명 사용자에게 설치 유도도 어려움). GPS 백그라운드 추적과 같은 "웹에서 반쯤 부서진 카드"에 핵심을 거는 실수를 반복하지 않는다.

따라서 재참여는 **능동적 알림이 아니라, 사용자가 앱을 다시 열었을 때**에 건다: ① 미완료 계획(Pending Plan) 직면 — 재방문 시 첫 화면에서 "어제 만든 계획, 하셨나요?"로 완료를 1탭에 닫게 함, ② 연속 기록(Streak) — 다시 열었을 때 보상·압박을 주어 끊고 싶지 않게 함. 웹 푸시/PWA는 후순위 향상 레이어로 둔다.

## Considered Options

- **푸시 리마인드(거부)** — 능동적으로 부를 수 있으나 웹/익명 환경에서 반쯤 부서짐. iOS 제약·설치 유도 난이도.
- **재방문 시점 직면 + Streak(채택)** — 능동적으로 부르진 못하지만, 돌아오기만 하면 완료를 쉽고 닫고 싶게 만든다. 푸시 불필요, 익명 쿠키로 동작.
