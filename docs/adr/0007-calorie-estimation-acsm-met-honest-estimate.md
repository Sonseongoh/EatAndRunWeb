# 칼로리·운동량은 ACSM MET 표준으로 추정하되 "정직한 추정"으로 다루고, 정확도는 사후 측정에서 얻는다

음식 분석 결과로부터 "목표 칼로리를 태우려면 어떤 방식으로 몇 분 운동"을 계산해야 한다. 계산은 **ACSM 표준 MET 대사 방정식**(`kcal/min = MET × 3.5 × 체중 / 200`)으로 하고, MET 값은 Compendium of Physical Activities 표준값을 쓴다(걷기 3.5·빠른걸음 5·달리기 9).

핵심 판단: **뛰기 전 시점엔 사용자의 실제 속도를 알 수 없어 사전 계산은 본질적으로 추정치**다. 페이스를 입력받아도 본인이 자기 페이스를 모르므로 GIGO다. 따라서 모드별 고정 MET가 오히려 정직한 기본값이며, 사전 공식을 더 정교화(페이스 기반 MET 등)하는 데 투자하지 않는다. 대신 **진짜 정확도는 사후 GPS 실측**(실제 거리·속도)에서 온다고 보고 그 방향으로 간다.

부수 결정: 칼로리(MET)와 경로 소요시간(페이스)이 서로 다른 속도를 가정하던 내부 불일치를 발견해, **둘 다 8.4km/h(달리기 MET 9 ↔ 기본 페이스 7.1분/km)로 통일**했다.

## Considered Options

- **ACSM MET 고정값 + "추정치" 인정(채택)** — 표준·정직·단순. 측정 불가능한 사전 시점에 합리적.
- **페이스 기반 MET(거부)** — 페이스 입력 자체가 추측값이라 더 정교해도 결국 추측(GIGO).
- **경사(고저차) 반영(보류)** — 정확도·차별화에 좋지만 Elevation API 비용. 검증 후로.
