# 경로 추천은 왕복(out-and-back) 기준으로 한다

경로 추천은 "목표 칼로리를 태울 거리"를 코스로 제시한다. 기존 구현은 반환점을 목표거리의 절반에 찍어놓고도 표시 거리·칼로리·시간은 *편도* 도로거리 기준이라, 사용자가 목적지에 남고 표시값이 목표와 어긋났다(~0.65배).

사용자는 출발지(집)에서 시작해 **돌아와야** 하고, "Y kcal 태우기"의 성공은 실제로 뛰는 *총 거리*로 재야 한다. 따라서 **왕복 기준으로 통일**한다: 반환점을 목표거리 절반에 두되 직선→도로 보정(휴리스틱 ~1.3)을 적용하고, 표시 거리·칼로리·시간·길 안내(출발→반환점→출발)를 전부 왕복 총량으로 맞춘다. 그 결과 "목표 kcal = 실제 왕복 거리"가 성립한다.

## Considered Options

- **왕복(채택)** — 출발지로 복귀, 목표 칼로리와 일치. 캐주얼 러닝의 자연스러운 모델.
- **편도(거부)** — 사용자를 목적지에 버려두고 표시값이 목표와 어긋남. 출퇴근런 같은 틈새 용도.
- **사용자가 왕복/편도 선택(보류)** — UI·로직 복잡도↑, 편도 수요 작음. 검증 후 옵션으로.
- **직선 배치 정밀 보정(보류)** — 첫 라우팅 결과로 반환점 재조정하면 정확하나 API 호출 2배(비용·지연).
