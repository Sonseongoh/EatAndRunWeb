# Eat & Run

먹은 것을 "지금 당장의 구체적인 행동"으로 바꿔, 사용자가 실제로 그 운동을 **하게** 만드는 건강 루틴 서비스. 음식 분석은 입구일 뿐이고, 제품의 가치는 그 행동을 끝까지 완수시키는 데 있다.

## Language

**Bridge (다리)**:
이 제품의 핵심 가치 명제. "방금 먹은 것 → 지금 당장 어떻게 태울지에 대한 구체적 행동"으로 잇는 연결. 음식을 *측정*하는 데서 멈추지 않고 *실행*까지 데려가는 것.

**Plan (계획)**:
음식 분석 결과와 그로부터 산출된 권장 운동(방식·시간·경로)의 묶음. 입구(`/analyze` → `/activity` → `/map`)에서 생성되며, 다리의 한쪽 끝이다. 현재 `history_entries`에 저장되는 "기록"은 전부 이 Plan에 해당한다.
_Avoid_: 기록(record) — 모호하므로 Plan/Completion으로 분리해 부른다.

**Completion (완수)**:
사용자가 Plan의 운동을 실제로 수행했다는 사실. 다리의 반대쪽 끝이며, **현재 데이터 모델에 존재하지 않는다**(아직 미구현). 제품의 진짜 성공은 여기서 측정된다.

**완수율 (Completion Rate)**:
이 제품의 핵심 성공 지표. 완수 ÷ (완수 + 놓침)으로 계산하며, 아직 결판나지 않은 오늘의 미완료(Pending)는 분모에서 제외한다("이미 결판난 계획 중 완수 비율"). 기본은 전체 기간 누적.

**미완료 계획 (Pending Plan)**:
생성 당일이며 아직 Completion으로도 Missed로도 닫히지 않은 Plan. 재방문 시 사용자가 가장 먼저 직면하는 대상이며, 여기서 완료를 1탭으로 닫는다. 직면 대상은 **오늘 생성된 Plan만**이다.

**놓침 (Missed)**:
생성된 당일 자정까지 완수되지 않아 닫힌 Plan. 다리의 본질이 "방금 먹은 걸 오늘 태운다"이므로, 지난 Plan은 영원히 묻지 않고 Missed로 마감해 통계로만 남긴다. (Plan은 Pending → Completed 또는 Missed 로 끝난다.)

**연속 기록 (Streak)**:
"그날 생성된 Plan을 하나라도 완수한 날"의 연속 일수. 만들고 안 한 날(또는 안 만든 날)이 있으면 끊긴다. 앱을 다시 열었을 때 작동하는 동기 후크. 매일 먹고-태우는 루프를 닫았다는 뜻이라 제품 가치(다리)와 정확히 일치한다.
